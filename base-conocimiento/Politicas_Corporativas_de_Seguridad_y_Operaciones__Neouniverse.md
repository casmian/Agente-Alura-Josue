# Políticas Corporativas de Seguridad y Operaciones — Neouniverse

Este documento reúne las políticas corporativas, directrices operativas y código de conducta tecnológica de Neouniverse. Es de cumplimiento obligatorio para todo el personal y contratistas asociados.

---

## 1. Políticas de Seguridad de la Información

En Neouniverse la seguridad de los datos de nuestros clientes es la prioridad número uno. Se definen las siguientes reglas de cumplimiento:
* **Autenticación Obligatoria**: Todo acceso a los entornos de AWS, OCI y código fuente debe requerir autenticación multifactor (MFA).
* **Rotación de Credenciales**: Las llaves de API, contraseñas de bases de datos y accesos personales deben rotarse cada 90 días.
* **Uso de VPN**: Se prohíbe el acceso a entornos productivos o consolas de administración desde redes públicas sin conexión VPN autorizada.



## 2. Código de Conducta Profesional

Fomentamos una cultura de respeto, colaboración constructiva y aprendizaje continuo.
* **Revisiones de Código**: Todas las solicitudes de extracción (Pull Requests) deben ser revisadas y aprobadas por al menos un Tech Lead.
* **Mentoría de Onboarding**: Los ingenieros senior deben actuar de forma socrática y constructiva al evaluar el código de nuevos ingresos, promoviendo el crecimiento sin dar soluciones directas.



## 3. Canales de Comunicación Oficiales

Para la coordinación del equipo técnico, los canales oficiales en Slack son:
* `#soporte-ti`: Para solicitudes de hardware o accesos.
* `#notificaciones-despliegue`: Canales automatizados de integración continua.
* `#incidentes-sev1`: Exclusivo para gestión de incidentes críticos.
